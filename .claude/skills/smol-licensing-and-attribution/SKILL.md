# Smol Licensing & Attribution

## Smol's own license: AGPL-3.0

Smol is AGPL-3.0. The `LICENSE` file (generated by GitHub at repo creation) is
authoritative. Do not modify it.

AGPL-3.0 requires:
1. Source code must be made available to users who interact with the software over a
   network (not applicable here — desktop app, no network service).
2. Any modifications distributed must also be AGPL-3.0.
3. The license and copyright notice must be preserved.

## FFmpeg codec whitelist (LGPL-safe)

Use ONLY these encoders to stay in the LGPL build:
- Video: `libx264` (LGPL), `h264_nvenc`, `h264_qsv`, `h264_amf`, `hevc_nvenc`, `libsvtav1`
- Audio: `libmp3lame` (LGPL), `aac` (built-in), `libopus` (BSD), `flac` (BSD)

Do NOT use: `libfdk_aac` (nonfree), `libx265` (GPL) in a GPL-incompatible context.

## Adding a new dependency checklist

1. Check license: MIT / Apache-2.0 / BSD / LGPL = generally fine. GPL = needs review.
2. Add an entry to `THIRD_PARTY_NOTICES.md`.
3. If the license requires attribution (IJG, BSD-4-clause, etc.), add the attribution
   text to the NOTICES entry.
4. Update this skill file's table above.
