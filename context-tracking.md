# 📚 Smol (formerly Squeeze) - Context & Alexandria Library

This document serves as the master brain, context tracker, and historical record for the **Smol** project. If the project ever burns down, or if context is lost, reading this file from top to bottom will restore full understanding of the goals, the architecture, the failures, and the exact current state.

## 🌟 1. Project Vision & Origin
**Problem:** In Egypt, internet bandwidth is heavily throttled and expensive (quotas are low). Uploading a 600MB video to Telegram/WhatsApp takes forever and eats the quota.
**Solution:** A local, offline, extremely fast, and visually beautiful compression tool that wraps FFmpeg. It takes a 600MB video and squeezes it to 30MB with virtually zero perceptible quality loss.
**Vibe:** "Vibe-coded." It should not look like a developer tool. It should feel premium, smooth, dark-mode-native, with micro-animations and zero complex configuration. You drop a file, it compresses.

## 🛠️ 2. The Tech Stack
* **Core Framework:** Tauri v2 (Rust + Webview)
* **Frontend:** React 19, TypeScript, Vite
* **State Management:** Zustand (Strictly enforced atomic selectors)
* **Styling & UI:** Tailwind CSS, Shadcn UI, Framer Motion (animations), Sonner (toasts), Lucide React (icons).
* **Typography:** Inter (UI text) & JetBrains Mono (numbers, codecs, technical data).
* **Engines (Sidecars):**
  * `ffmpeg` / `ffprobe` (for video, audio, image thumbnailing and compression)
  * `ghostscript` (for PDF compression - planned)

## ⚖️ 3. The Hard Rules (Lessons from the "Squeeze" Post-Mortem)
The first attempt at this app ("Squeeze") failed due to structural regressions. To prevent this, the project operates under strict "Hard Rules" (HR) to prevent history from repeating:

* **HR-4 (Window Chrome):** Never use `transparent: true` for the Tauri window. It breaks native Windows drag-and-drop and introduces rendering artifacts. We use `decorations: true` and build a custom HTML titlebar.
* **HR-5/6 (Rust FileSystem Bridge):** The frontend JS must **never** call filesystem APIs directly. All `fs` operations (stat, read, write) must be routed through safe Rust `#[tauri::command]` functions to prevent Tauri sandbox scope crashes.
* **HR-7 (Zustand Atomic Selectors):** Zustand selectors must return primitives (`number`, `boolean`, `string`) or stable state references. **Never** return a dynamically created array/object (e.g., `Object.values(s.jobs)`) as it breaks referential equality and causes infinite React re-render loops (`Maximum update depth exceeded`).
* **HR-11 (Tauri Asset Protocol):** Local images/thumbnails generated by Rust in temporary folders cannot be rendered in the browser using raw `C:\...` paths. They must be wrapped in Tauri's `convertFileSrc(path)` from `@tauri-apps/api/core` to bypass strict CSP restrictions.
* **HR-15 (Native Drag and Drop):** HTML5 `onDrop` events do not expose absolute file paths in Tauri due to browser security. All file dropping must be handled via Tauri's native `tauri://file-drop` events.

## 📝 5. Bug / Regression Log (The Graveyard)
* **Bug #1 (The Loop of Death):** `QueueTotalBanner` called `useJobsStore(s => Object.values(s.jobs))`. It caused an infinite loop and black screen. *Fixed in Phase 3.5 by migrating to Pattern C (primitive-returning selector loops).*
* **Bug #2 (The Ghost Drop):** HTML5 `onDrop` used instead of Tauri native drop. Hover worked, dropping did nothing because absolute paths weren't passed. *Fixed in Phase 3.6.*
* **Bug #3 (The Blind App):** Thumbnails returned from Rust were blocked by Chromium CSP because they weren't wrapped in `convertFileSrc`. *Fixed in Phase 3.6.*
* **Bug #4 (The Double Event):** React Strict Mode mounted the Tauri `onDragDropEvent` listener twice because it lacked a cleanup function, causing duplicate file drops. *Fixed in Phase 3.7.*
* **Bug #5 (The UI Freeze):** Generating thumbnails blocked the main thread because the Rust `#[tauri::command]` was synchronous. *Fixed in Phase 3.7 by making the commands `async`.*
